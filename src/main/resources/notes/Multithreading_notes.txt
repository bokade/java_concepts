üß© 1. What is a Thread?

üëâ A thread is the smallest unit of execution inside a program.
Each Java program runs at least one main thread (the one that runs your main() method).

üí° You can think of:

Process = an independent running program (e.g., chrome.exe, java.exe)

Thread = a lightweight subprocess inside a process

Each process can have multiple threads sharing the same memory.

üß† Example analogy:

Imagine a restaurant:

Process ‚Üí The restaurant

Threads ‚Üí The waiters inside the restaurant
All waiters (threads) work in parallel but share the same kitchen (memory).

üß© 2. Why Multithreading?

‚úÖ Advantages:

Better CPU utilization (tasks run in parallel)

Better performance in I/O operations (e.g., file read + database call)

Used in real-world systems:

Web servers

Banking systems

Gaming / UI applications

‚ùå Disadvantages:

Synchronization issues (data corruption if not managed properly)

üß© 3. Thread Lifecycle (Important for interviews)

A Java thread has several states (from Thread.State enum):

State	Meaning
NEW	Thread object created but start() not called
RUNNABLE	Thread ready to run or running
BLOCKED/WAITING	Waiting for lock or other thread
TERMINATED (DEAD)	Thread finished execution

Lifecycle looks like this:

NEW ‚Üí RUNNABLE ‚Üí RUNNING ‚Üí WAITING/BLOCKED ‚Üí TERMINATED

Thread Scheduling & Priority (Quick Overview)

Java thread scheduler decides which thread runs next.

Priority range: Thread.MIN_PRIORITY (1) ‚Üí Thread.MAX_PRIORITY (10)
Default = 5

Higher priority threads may get more CPU time, but not guaranteed (depends on OS).

t1.setPriority(Thread.MAX_PRIORITY);
t2.setPriority(Thread.MIN_PRIORITY);


ThreadGroup (Basic Awareness)

A ThreadGroup is used to group multiple threads together for easier management. Rarely used now since ExecutorService replaced it.

Example:

ThreadGroup group = new ThreadGroup("MyGroup");
Thread t1 = new Thread(group, new MyRunnable(), "T1");
Thread t2 = new Thread(group, new MyRunnable(), "T2");
